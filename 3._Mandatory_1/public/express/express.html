express

- Express er et node web framework, som gør det let at arbedje med HTTP i node.
- Vi laver en express variable, som vi sætter express bibloteket til.
- Derefter laver en app variable, som vi sætter værdien express() functionen.
- Kan lave det en oneliner, men hvis vi skal benytte .use, kræver det vi har sat app værdien til express().

app.listen

- .listen giver os mulighed for at sætte den port vi vil køre vores server på.
- Man kan godt bare skrive porten, men det er meget rart, at havde en lille function, der siger i consolen hvad port vi køre på.

app.use

- app.use(middleware) er noget der bliver kaldt, hver gang der er en requests til serveren. Vi har brugt app.use til express.json og express.static
- app.use(express.json()), parser indgående requests om til JSON.
- app.use(express.static('mappeNavn')), er en sikkerhedsfeature, der gør at en browser kun kan se de filer man har givet lov til.

app.get

- .get er en http get request
- Bruges til f.eks. at sende informationer til client, kan være html filer, json objekter osv..
- Kan sende JSON object baseret på ID, eller sende et helt array
- Hvis vi skulle sende filer igennem .get, ville vi skrive res.sendFile(__dirname + "/path")

app.patch

- .patch er en http patch request
- Bruges til at update en del af ens JSON object.
- F.eks. hvis vi vil updatere en memes, kan vi finde en memes vi vil update med ID, og derefter ændre den meme uden at ændre ID.

app.post

- .post er en http post request
- Bruges til at create JSON object(primært)
- Får automatisk givet et ID, så skal kun give de informationer der skal være i objektet.

app.delete

- .delete er en http delete request
- Bruges til at delete JSON object
- Vi tager et ID parameter, som vi så laver et loop på, for at finde den rigtige memes, som så bliver slettet fra vores array.

Res & Req

- req står for request
- res står for response
- req er et object der indeholder alt information om den HTTP request der er blevet lavet.
- res kan vi bruge til at sende data til client
